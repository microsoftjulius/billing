Expected Behavior

✅ Active State: The clicked navigation item should have a blue background (primary color)

✅ URL Change: The browser URL should change to match the route

✅ Page Content: The main content area should load the corresponding component

✅ Breadcrumb: The breadcrumb should update to show the current page

CSS Classes Applied

We've implemented multiple fallback mechanisms:

  • active-class="nav-link--active" - Vue Router active class
  • exact-active-class="nav-link--active" - Vue Router exact active class
  • :class="getNavLinkClass('/app/route')" - Manual computed class
  • .router-link-active - Default Vue Router class
  • .router-link-exact-active - Default Vue Router exact class

Troubleshooting

If navigation isn't working:

  1. Check browser console for JavaScript errors
  2. Verify you're accessing the Laravel app (port 8000) not Vite dev server (port 5173)
  3. Check if the CSS variables are loaded properly
  4. Inspect the navigation elements to see if classes are being applied
